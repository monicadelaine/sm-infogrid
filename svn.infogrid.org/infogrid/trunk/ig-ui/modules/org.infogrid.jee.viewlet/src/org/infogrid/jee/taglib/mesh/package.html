Tags to display MeshObjects and their various properties and relationships.

Tags to display times and durations related to MeshObjects

MeshObject creation
{@link org.infogrid.jee.taglib.mesh.MeshObjectTimeCreatedTag MeshObjectTimeCreatedTag} renders the time at which the MeshObject was created; {@link org.infogrid.jee.taglib.mesh.MeshObjectSinceTimeCreatedTag MeshObjectSinceTimeCreatedTag} renders the duration since the MeshObject was created.
Last modification of a MeshObject
{@link org.infogrid.jee.taglib.mesh.MeshObjectTimeUpdatedTag MeshObjectTimeUpdatedTag} renders the time at which the MeshObject was last updated; {@link org.infogrid.jee.taglib.mesh.MeshObjectSinceTimeUpdatedTag MeshObjectSinceTimeUpdatedTag} renders the duration since the MeshObject was last updated.
Last read access to a MeshObject
{@link org.infogrid.jee.taglib.mesh.MeshObjectTimeReadTag MeshObjectTimeReadTag} renders the time at which the MeshObject was last read; {@link org.infogrid.jee.taglib.mesh.MeshObjectSinceTimeReadTag MeshObjectSinceTimeReadTag} renders the duration since the MeshObject was last read.

The following tag attributes are provided on the tags that render a time stamp:

Attribute Meaning Required?
meshObjectName Name of the bean whose properties are rendered. Must be a MeshObject. required
timeZone Time zone for the rendered time. This defaults to the current time zone. This String must be given in a format that can be parsed by java.util.TimeZone.getTimeZone(String). optional
locale Locale for the rendered time, such as en-us. This defaults to the current locale. optional
dateStyle Style for the date portion of the time stamp (see java.text.DateFormat). One of FULL, LONG, MEDIUM, and SHORT. optional
timeStyle Style for the time portion of the time stamp (see java.text.DateFormat). One of FULL, LONG, MEDIUM, and SHORT. optional
filter If true, filter output for characters that are sensitive in HTML. optional
ignore If true, ignore missing beans and simply output nothing. Otherwise, when beans are missing, throw a JspException optional
scope The scope (page, request, session, application) in which beans are being looked up optional

Tags to display the type(s) of a MeshObject

The {@link org.infogrid.jee.taglib.mesh.MeshObjectBlessedByIterateTag MeshObjectBlessedByIterateTag} iterates over all MeshTypes with which the MeshObject is blessed. It does not perform any rendering itself, but enables other contained tags to render the found MeshTypes appropriately.

The following tag attributes are provided:

Attribute Meaning Required?
meshObjectName Name of the bean whose types are being iterated over. Must be a MeshObject. required
blessedByLoopVar Name of the bean that will be set as the iteration variable with the EntityType. required
filter If true, filter output for characters that are sensitive in HTML. optional
ignore If true, ignore missing beans and simply output nothing. Otherwise, when beans are missing, throw a JspException optional
scope The scope (page, request, session, application) in which beans are being looked up optional

The {@link org.infogrid.jee.taglib.mesh.MeshTypeTag MeshTypeTag} enables the rendering of a MeshType in a consistent manner.

The following tag attributes are provided:

Attribute Meaning Required?
meshTypeName Name of the bean that is the MeshType to be rendered. Must be a MeshType. required
nullString String to be displayed if the PropertyValue is null. optional
maxLength Maximum length, in characters, of the rendered text. optional
locale If the specified property is of type L10Map, this attribute specifies the locale in which the value shall be retrieved. If this is not given, the default locale is assumed. This attribute is ignored for all properties that are not L10Maps. Locales are provided in the form NN-NN-NN, NN-NN or NN, e.g. en and de-at. optional
exactLocaleMatch If attribute locale is given, this boolean value indicates whether an exact locale match is required, or whether a less specific locale is also applicable (e.g. variants vs. country vs. language). optional
filter If true, filter output for characters that are sensitive in HTML. optional
ignore If true, ignore missing beans and simply output nothing. Otherwise, when beans are missing, throw a JspException optional
scope The scope (page, request, session, application) in which beans are being looked up optional

The {@link org.infogrid.jee.taglib.mesh.MeshObjectPropertyIterateTag MeshObjectPropertyIterateTag} iterates over all, or a named set of properties of a MeshObject.

The following tag attributes are provided:

Attribute Meaning Required?
meshObjectName Name of the bean whose properties are being iterated over. Must be a MeshObject. required
meshTypeName Name of the bean to whose properties the iteration is restricted. This is optional. if not given, this iterator iterates over all properties of the MeshObject; if given, only the properties of that EntityType are being considered. Must be an EntityType. optional
propertyList List of property names to iterator over, separated by listSeparator (","). optional
propertyValueLoopVar Name of the bean that will be set with the current PropertyValue inside the loop. optional
propertyTypeLoopVar Name of the bean that will be set with the current PropertyType inside the loop. optional
listSeparator The separator String by which the entries in the propertyList are separated. The default is a comma (","). optional
skipNull If true, properties with null PropertyValues are skipped in the iteration. optional
ignore If true, ignore missing beans and simply output nothing. Otherwise, when beans are missing, throw a JspException optional
scope The scope (page, request, session, application) in which beans are being looked up optional

The {@link org.infogrid.jee.taglib.mesh.MeshObjectLinkTag MeshObjectLinkTag} creates the HTML for a hyperlink to a MeshObject. Clicking on the generated link will send the browser to the respective MeshObject.

Example:

<MeshObjectLinkTag meshObjectName="obj" />

will look up a MeshObject added as attribute named 'obj' to the (page context), prepending the http://example.com/foo/ address the local replica.

The following tag attributes are provided:

Attribute Meaning Required?
meshObjectName Name of the bean whose types are being iterated over. Must be a MeshObject. required
addArguments String with additional arguments to be appended to the constructed URL. This should be of the form a=b&c=d&e=f. optional
filter If true, filter output for characters that are sensitive in HTML. optional
ignore If true, ignore missing beans and simply output nothing. Otherwise, when beans are missing, throw a JspException optional
scope The scope (page, request, session, application) in which beans are being looked up optional

Tags to display a MeshObject's related MeshObjects

The {@link org.infogrid.jee.taglib.mesh.MeshObjectNeighborIterateTag MeshObjectNeighborIterateTag} iterates over the neighbor MeshObjects of a given MeshObject (corresponding to method MeshObject.traverseToNeighbors()).

The following tag attributes are provided:

Attribute Meaning Required?
meshObjectName Name of the bean over whose neighbors we iterate. Must be a MeshObject. required
neighborLoopVar Name of the bean that contains the iteration (current neighbor) MeshObject. required
ignore If true, ignore missing beans and simply output nothing. Otherwise, when beans are missing, throw a JspException optional
scope The scope (page, request, session, application) in which beans are being looked up optional

The {@link org.infogrid.jee.taglib.mesh.MeshObjectRelatedIterateTag MeshObjectRelatedIterateTag} iterates over the MeshObjects found by traversing from a given MeshObject via a given TraversalSpecification (corresponding to method MeshObject.traverse( TraversalSpecification )).

The following tag attributes are provided:

Attribute Meaning Required?
meshObjectName Name of the bean whose that is the start of the traversal. Must be a MeshObject. required
traversalSpecificationName External form of the TraversalSpecification to traverse. required
relatedLoopVar Name of the bean that contains the iteration (destination) MeshObject. required
ignore If true, ignore missing beans and simply output nothing. Otherwise, when beans are missing, throw a JspException optional
scope The scope (page, request, session, application) in which beans are being looked up optional

The {@link org.infogrid.jee.taglib.mesh.MeshObjectRoleIterateTag MeshObjectRoleIterateTag} iterates over the RoleTypes that the start MeshObject plays with respect to a destination MeshObject.

ignore If true, ignore missing beans and simply output nothing. Otherwise, when beans are missing, throw a JspException optional
scope The scope (page, request, session, application) in which beans are being looked up optional
